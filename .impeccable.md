## Design Context

### Users
Casual sports fans and watch party groups who want a free, social bingo game layered on top of live sports. Users are watching games at home, at bars, or with friends — the app is a second-screen companion that makes any game more engaging. Players will eventually work towards winning rewards, so the experience should feel exciting and worth coming back to. The core job: make watching sports more fun by turning game moments into a shared competition.

### Brand Personality
**Fun, scrappy, irreverent.** Dobber has underdog energy — it's playful, doesn't take itself too seriously, and has real personality. It's the friend at the watch party who makes everything more entertaining. The tone should feel like friendly trash talk, not a corporate sportsbook.

### Aesthetic Direction
- **Visual tone**: Arcade scoreboard meets mobile-first sports gaming. Deep navy-black backgrounds (#0c0c14) with burnt orange (#ff6b35) as the signature accent. Monospace typography (JetBrains Mono) for UI gives a technical/scoreboard feel; Bebas Neue for display creates bold, punchy headlines.
- **Theme**: Dark mode only (light mode disabled). Glowing orange accents, beveled card surfaces, machine-frame styling.
- **References**: PrizePicks/Underdog (clean dark cards, orange accents, mobile-first sports entries), Jackpocket/lottery apps (gamified, reward-driven, fun animations and reveals).
- **Anti-references**: Avoid looking like a traditional sportsbook (ESPN Bet, DraftKings) — too dense, too serious. Avoid corporate/sterile SaaS aesthetics. Avoid cluttered data-heavy interfaces.
- **Animation philosophy**: Rich micro-interactions that feel alive — splash screen ball roll-in, card stagger entrances, mark pops, win ring pulses, confetti. Motion should feel spring-loaded and satisfying, never sluggish.

### Design Principles
1. **Arcade energy, mobile polish** — Every interaction should feel like tapping a machine button: responsive, satisfying, slightly theatrical. But the layout and spacing should be clean and mobile-native, not retro for retro's sake.
2. **Celebration over information** — Prioritize moments of delight (marking a square, hitting bingo, climbing the leaderboard) over dense data displays. The fun is in the game, not the stats.
3. **Scrappy confidence** — Design should feel bold and opinionated, not safe. Use the burnt orange generously. Let personality show in copy, animations, and interactions. Punch above your weight.
4. **Second-screen simplicity** — Users are watching a game, not studying an app. Keep cognitive load low. Big touch targets, clear states, glanceable information. The board should be readable from across the room.
5. **Reward the return** — Design patterns should reinforce progression and reward. Make coming back feel worthwhile through visual feedback, streaks, and satisfying state changes.

### Technical Constraints
- **Stack**: React 19, Vite, Tailwind CSS v4, Supabase, Capacitor (iOS native), Stripe, PWA
- **Design tokens**: Full CSS variable system in `src/styles/dobber-brand.css` with `--db-*` prefix
- **Tailwind bridge**: Theme tokens mapped in `src/styles/globals.css` via `@theme` block
- **Fonts**: Bebas Neue (display/headings), JetBrains Mono (body/UI/mono)
- **Accessibility**: WCAG AA, `prefers-reduced-motion` respected, orange focus rings, 44px iOS touch targets
- **iOS-first**: Safe area support, Capacitor native shell, touch-optimized interactions
